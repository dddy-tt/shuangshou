# 设计目标 vs 代码实现 差异表

生成日期：2026-06-29  
版本：v3.1  
依据：`docs/current-pin-matrix.md`、`shuangshou.ioc`、当前代码

## 已一致

| 项目 | 设计目标 | 当前实现 | 状态 |
|---|---|---|---|
| ADC 右手 5 路 | `PA0..PA4 / ADC1` | `PA0..PA4 / ADC1` | 一致 |
| ADC 左手 5 路 | `PA5..PA7 + PB0 + PB1 / ADC2` | 相同 | 一致 |
| 软 I2C 三路 | `PB6/7`、`PB8/9`、`PC6/7` | 相同 | 一致 |
| DFPlayer 串口 | `USART1 PA9/PA10` | 相同 | 一致 |
| 蓝牙串口 | `USART3 PC10/PC11` | 相同 | 一致 |
| TIM3 调度时钟 | 1ms 中断 | 1ms 中断 | 一致 |
| 手势状态机 | 五态状态机 | 五态状态机 | 一致 |
| 模式数量 | 翻译 / 控制 / 康复 | 3 种模式 | 一致 |

## 不一致

| 问题 | 代码实际 | 文档写法 | 影响 | 建议 |
|---|---|---|---|---|
| 振动 PWM 引脚 | `PD12`、`PD13` 两路 | `PE2..PE5` 四路 | 硬件接线参考会冲突 | 先改文档，不先改代码 |
| 振动通道数 | 2 路 | 4 路 | 设计和实现不一致 | 先确认真实需求 |
| 蓝牙模块型号 | 代码目标应配 `JDY-33` | 现有硬件仍可能是 `JDY-31` | Web Bluetooth 不兼容 | 采购 `JDY-33` |

## 注释和文档过时

| 问题 | 文件 | 说明 | 建议 |
|---|---|---|---|
| `MPU6050` 注释残留 | `soft_i2c.c`、`soft_i2c.h` | 当前实际已迁移到 `JY61P` | 仅改注释 |
| 时钟口径混用 | `gesture.c` | 旧版用过 `HAL_GetTick()` | 统一到 `sys_tick_ms` |
| `.ioc` 未体现 `PE0` | `.ioc` | 软串口由代码手动初始化 | 文档说明即可 |

|---|---|

## 结论

当前最需要保持的原则是：

1. 以 `.ioc` 和当前代码作为“已实现事实”。
2. 以设计文档作为“目标方案”，不要直接覆盖代码事实。
3. 在硬件未到位前，只做低风险修正，不做引脚重映射。
