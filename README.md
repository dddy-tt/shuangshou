# 面向失语症患者与老年护理的智能手语交互手套 — v2.4 工业防御级方案

> **版本**：v2.4（工业防御级 — 7 大硬件暗桩 + 4 项次生高阶暗桩定点清除）
> **v2.0 摘要**：PA2/PA3 引脚冲突、全线模拟 I2C、MAX30102 供电抗干扰、门限微分法轨迹检测。
> **v2.1 摘要**：MAX30102 400kHz 逐样本读取；振动马达 SI2302 MOS 驱动 + 1N4148 续流；DFPlayer DMA 非阻塞发送。
> **v2.2 摘要**：电源树重构 — MT3608 直供主干(MCU+DFPlayer,耐600mA+峰值)，MAX30102 独立 LC 分支隔离音频噪声。
> **v2.3 摘要**：7 大防御性重构 — soft_i2c 超时熔断 (2000次)；gesture 旋转熔断+冷启动保护+百分比钳位；main ADC DMA 快照防撕裂；MAX30102 100Hz 对齐 FIFO 溢出复位+IR[5000,200000]边界卡关。
> **v2.4 摘要**：4 项次生暗桩清除 — JY61P 200Hz I2C 速率锁定 (根除 Jerk ≡0 伪零点)；MAX30102 while 突发清仓 (防渐进式 FIFO 积压)；ADC Ping-Pong 双缓冲快照 (删除 DMA Stop/Start 高频切流)；gesture 250ms 双手异步松弛锁存窗 (解决生理异步漏判)。

## 1. 项目概述

### 1.1 背景

在合泰杯单手智能手套（STM32F103C8T6 → HT32F52352）基础上，升级为**基于 STM32F407ZGT6 的双手中文字语翻译系统**，用于嵌入式大赛 ST 赛道。

**应用定位**：面向失语症患者术后交流、老年护理场景的穿戴式手语翻译 + 生命体征监测设备。

### 1.2 真实手语能力边界（技术诚实）

真实中国手语（CSL）单靠柔性传感器 + IMU **做不全**，原因如下：

| 手语维度 | 本方案能否感知 | 原因 |
|----------|:---:|------|
| **手形**（手指怎么弯） | ✅ 能 | 10 路柔性传感器，每指 3 态（伸直/半弯/全弯）识别 |
| **手掌朝向**（掌心朝哪） | ✅ 能 | JY61P 卡尔曼滤波输出 Pitch/Roll/Yaw（三轴全有，零漂移），航向角可用 |
| **运动方向**（手往哪动） | ⚠️ 半能 | 门限微分法判方向（上/下/左/右/前/后），判不出复杂轨迹形状（圆 vs 折线） |
| **相对身体位置**（手在额头/下巴/胸前） | ❌ 不能 | 纯 IMU 无绝对位置参考。如"爸爸"（手在额头）和"妈妈"（手在下巴）手形相同，无法区分 |
| **面部表情**（手语语法中的疑问/否定/强调） | ❌ 不能 | 需要摄像头 + 计算机视觉 |

> **诚实定位**：本方案能做 **40~60 个手势**（手形 + 方向组合），覆盖日常交流 + 医护场景常用语。承诺"全手语翻译"是 CV + 深度学习的研究级课题，不是嵌入式大赛该走的路。

### 2.2 关键设计决策（针对专家退修意见的回应）

#### 决策 A：蓝牙移至 USART3（PC10/PC11），释出 PA2/PA3

原方案将蓝牙挂在 USART2（PA2/PA3），但 PA2/PA3 同时也是 ADC1 的 CH2/CH3，右手食指和中指的传感器必须用这两个通道。通电后 ADC 采样与 UART 同时驱动同一引脚，电压冲突导致二者均不可用。

**修正**：蓝牙移至 USART3（PC10=TX, PC11=RX），两条引脚纯净无复用冲突。PA0~PA4 完整留给 ADC1（右手五指）。

#### 决策 B：全线改用模拟（软件）I2C

原方案使用 STM32 硬件 I2C 外设。在 1 米排线 + 双手运动的真实穿戴场景下，线间寄生电容在 SCL 上升沿产生回勾，硬件 I2C 的状态机一旦检测到 SDA 在 CLK 高电平期间变化，会进入 BUSY 锁死状态，只能通过复位外设恢复。这在实时手势识别中不可接受。

**修正**：

1. **三路外设全部改用 GPIO 模拟 I2C**（软件翻转 SCL/SDA）。F407 主频 168MHz，5 条 NOP 即可满足标准模式 100kHz 时序，远超 JY61P 的需求。MAX30102 因需随主循环高频读取，**SoftI2C_3 独立升级为 400kHz 快速模式**（SCL 半周期 ≈ 1.25μs，168MHz 下 ≈ 1.25μs / 5.95ns ≈ **210 个 CPU 周期**，充裕的 GPIO 翻转裕量）。
2. **排线物理层**：每根软 I2C 排线采用 **SCL / GND / SDA 三线并排** 结构，GND 线夹在 SCL 和 SDA 之间提供电磁屏蔽。不使用传统的 SCL-SDA 双绞线（缺少地平面隔离，串扰严重）。
3. 软件 I2C 在 SCL 上升沿后延迟 2μs（100kHz 模式）/ 0.5μs（400kHz 模式）再读取 SDA，给足长线上升时间裕量。
4. **MAX30102 读取时序优化**：放弃"每 100ms 批量抓取 10 个 FIFO 样本"的集中阻塞方案（100kHz 下耗时 5.85ms，直接打穿 5ms JY61P 轮询周期）。改为 **"每 10ms 随主循环读取 1 个 FIFO 样本"**（400kHz 下耗时 ≈ 195μs），消弭了长阻塞，确保 5ms 硬实时任务不受干扰。

#### 决策 C：电源树重构 — 大电流直供 + MAX30102 独立 LC 滤波分支

原方案将 LC 滤波器串在主供电干线上。**这是物理错误**：DFPlayer Mini 驱动 3W 喇叭时峰值电流高达 600mA，叠加 F407 (~100mA) 和传感器 (~50mA)，总干线电流可达 750mA。10μH 色环电感（典型耐流仅 50~100mA，直流电阻 2~5Ω）串联在干线上必然导致电压骤降，MCU 在 DFPlayer 播报瞬间因欠压复位（BOR），Demo 直接死机。

**修正**：采用**干线直供 + 敏感支路独立滤波**的拓扑：

```
MT3608 输出 (5V, 含纹波 50~150mVpp)
  │
  ├──→ 5V 大电流干线 (直通, 无电感, 可承载 1A+)
  │      ├──→ DFPlayer Mini VCC (峰值 600mA, 音频大电流回路)
  │      └──→ AMS1117-3.3V (#1) → 3.3V 数字主干
  │              ├──→ STM32F407 VDD (~100mA)
  │              ├──→ JY61P × 2 VCC (~3.6mA × 2)
  │              ├──→ JDY-31 VCC (~10mA)
  │              └──→ 振动马达 (经 SI2302, 60mA × 2, 开关瞬态)
  │
  └──→ 10μH 电感 (串联) ─┬─ 100μF/16V 电解 ─┴─ GND   ← 【LC 低通, 仅 MAX30102 支路】
                          │    f_c ≈ 5kHz, @1.2MHz 衰减 ≈ 47dB
                          │    支路电流 < 5mA, 普通色环电感完全胜任
                          │
                          └──→ AMS1117-3.3V (#2) → 3.3V MAX30102 专用
                                   │
                                   ├─ 10μF 钽电容 (低 ESR) ─┴─ GND
                                   └─ 100nF 瓷片电容 ──────┘
```

**核心原则**：
- **大电流不穿电感**：DFPlayer 音频瞬态电流走 5V 干线直供回路，电感不在其路径上，电压不做功放级的"刹车"
- **敏感电路独享滤波**：MAX30102 支路电流 < 5mA，10μH 色环电感轻松承载，LC 滤波器有效衰减 1.2MHz 开关噪声 47dB，PPG 波形干净
- **物理隔离音频噪声**：DFPlayer 喇叭回路与 MAX30102 模拟前端在 PCB/面包板上分属两条独立电源树分支，共享点仅在 MT3608 输出电容处（低阻抗节点），低频音频电流不耦合到 PPG 信号链

#### 决策 D：动态轨迹算法重构 — 放弃积分，改用门限微分法

原方案 5.2 节提出的加速度双重积分方案存在根本性缺陷。当手腕倾斜时，重力加速度 1g 会投影到加速度计的 X、Y、Z 轴，而 JY61P 的姿态解算（Pitch/Roll）精度有限（±2°），残留的 1g 投影误差 ≈ 35mg。二次积分后：

```
位置误差 ≈ 0.5 × 35mg × t²
t=1s 时: 0.5 × 0.035 × 9.8 × 1² ≈ 0.17m = 17cm
t=2s 时: 0.5 × 0.035 × 9.8 × 4 ≈ 0.69m
```

2 秒后位置已完全失去意义，方向判断同样失效。

**修正**：采用**基于滑动窗口的加加速度（Jerk）门限法**，详见 §5.2。

> **⚡ 振动马达驱动电路说明**
>
> STM32 GPIO 最大输出电流仅 25mA，而 Φ10mm 振动马达额定电流约 60mA（启动瞬态更高）。直接用 GPIO 灌电流必然烧毁 PD12/PD13。
>
> **正确做法**（低端 N-MOS 开关）：
> ```
> VCC_3V3 ──┬── 振动马达 ──┬── SI2302 Drain
>           │              │
>        1N4148(反偏)      SI2302 Source → GND
>           │              │
>           └──────────────┘          SI2302 Gate ← PD12/PD13 (TIM4 PWM)
>
> 原理: GPIO PWM 高电平 → SI2302 导通 → 马达电流经 MOS 流入 GND
>       GPIO PWM 低电平 → SI2302 关断 → 马达线圈反向电动势经 1N4148 续流，不击穿 MOS
>       GPIO 仅驱动 MOS 栅极（输入电容 ~200pF），峰值电流 < 1mA，绝对安全
> ```

> **为什么选 DFPlayer Mini 而非 TTS 芯片？**
> 
> - TTS 芯片（SYN6658 / XFS5152）实时合成语音，音质机械、模块 ¥40+、接线复杂（SPI）
> - DFPlayer Mini（¥5）：真人有感情录音存 SD 卡，串口 `play(N)` 5 行代码搞定
> - 比赛演示优势：预录音质温暖自然，不会因 TTS 网络请求超时而卡顿
> - 自定义手势支持：手机录音 → 蓝牙下发 → F407 接收编号存入 SD 卡

## 5. 手势识别算法

### 5.1 手势编码格式

每个手势用 **11 位编码** 描述：

```
[L0][L1][L2][L3][L4] [R0][R1][R2][R3][R4] [T]

L0~L4  左手 5 指状态 (拇指 → 食指 → 中指 → 无名指 → 小指)
R0~R4  右手 5 指状态 (同上)
T      轨迹方向码

手指状态值:
  0 = 伸直 (弯曲度 < 40%)
  1 = 半弯 (弯曲度 40% ~ 75%)
  2 = 全弯 (弯曲度 > 75%)

轨迹方向值:
  0 = 静止（仅用静态手形判定，无显著运动）
  1 = 上挥        4 = 下压
  2 = 左挥        5 = 右挥
  3 = 前推        6 = 后拉

编码示例:
  "00000"+"11111"+"0"  左手平伸 + 右手握拳 + 静止 → "你好"
  "01000"+"01000"+"1"  双手食指半弯 + 向上挥 → "我需要帮助"
  "11111"+"00000"+"4"  左手握拳贴胸 + 下压 → "胸痛"
```

### 5.2 动态轨迹检测算法（门限微分法 — 专家退修后重构）

#### 5.2.1 为什么放弃加速度积分

加速度双重积分在穿戴式设备上是经典陷阱。手腕任何微小的倾斜（Pitch/Roll 解算误差 ±2°）都会导致重力加速度 1g 沿 X/Y/Z 轴泄露。残留的重力投影 ≈ 35mg，二次积分后 2 秒内位置漂移可超 50cm，方向判断完全失效。

#### 5.2.2 替代方案：基于滑动窗口的加加速度（Jerk）门限法

**核心思路**：不跟踪"手去了哪里"（积分），只检测"手是否在剧烈运动"以及"哪个轴的变化最剧烈"（微分）。

```
┌─────────────────────────────────────────────────────────┐
│  门限微分法 (Jerk-based Direction Detector)              │
│                                                          │
│  输入: 三轴加速度 ax, ay, az (原始值，不减去重力!)         │
│       三轴陀螺仪 gx, gy, gz                               │
│                                                          │
│  参数:                                                    │
│    WINDOW_SIZE = 20    (20 × 10ms = 200ms 滑动窗口)       │
│    JERK_THRESHOLD = 8000  (加加速度和的门限，需实测标定)    │
│    STABLE_THRESH = 1500  (静止判定门限)                   │
│                                                          │
│  数据结构:                                                │
│    int16_t ax_hist[WINDOW_SIZE]  加速度历史                │
│    int16_t ay_hist[WINDOW_SIZE]                           │
│    int16_t az_hist[WINDOW_SIZE]                           │
│    uint8_t hist_idx                                        │
│                                                          │
│  步骤 1: 滑动窗口维护 (每 10ms)                            │
│    读取 raw_accel_x/y/z 推入环形缓冲区                     │
│                                                          │
│  步骤 2: 计算三轴差分绝对值和 (Jerk Sum)                   │
│    jerk_x = ∑|ax[i] - ax[i-1]|   i=1..WINDOW_SIZE-1      │
│    jerk_y = ∑|ay[i] - ay[i-1]|                            │
│    jerk_z = ∑|az[i] - az[i-1]|                            │
│    jerk_total = jerk_x + jerk_y + jerk_z                  │
│    jerk_gyro  = ∑|gx[i]-gx[i-1]| + ∑|gy[i]-gy[i-1]|      │
│               + ∑|gz[i]-gz[i-1]|                          │
│                                                          │
│  步骤 3: 状态机判定                                        │
│    STATE_IDLE: jerk_total > JERK_THRESHOLD → STATE_MOVING │
│    STATE_MOVING: 记录三轴最大 jerk 的轴和方向:             │
│      dominant_axis = argmax(jerk_x, jerk_y, jerk_z)       │
│      direction_sign = sign(该轴净位移方向)                 │
│      jerk_total < STABLE_THRESH 持续 300ms → STATE_DONE   │
│    STATE_DONE: 输出方向编码 (1~6)                          │
│                                                          │
│  步骤 4: 方向编码映射                                      │
│    dominant_axis = X 且 sign>0 → 右挥  (编码 5)           │
│    dominant_axis = X 且 sign<0 → 左挥  (编码 2)           │
│    dominant_axis = Y 且 sign>0 → 上挥  (编码 1)           │
│    dominant_axis = Y 且 sign<0 → 下压  (编码 4)           │
│    dominant_axis = Z 且 sign>0 → 后拉  (编码 6)           │
│    dominant_axis = Z 且 sign<0 → 前推  (编码 3)           │
│    陀螺仪大幅旋转且加速度 jerk 小 → 旋转手势 (编码 7, 扩展)│
└─────────────────────────────────────────────────────────┘
```

**关键优势**：
1. **不依赖积分**，完全规避重力泄露导致的漂移
2. **天然对 DC 偏移免疫**：差分运算消去恒定的重力偏置，只有真正的手部加速度变化产生信号
3. **仅需整数运算**：绝对值求和 + 环形缓冲区，无浮点，168MHz 下 200 样本窗口运算耗时 < 50μs
4. **陀螺仪辅助**：区分"手平移"和"手旋转"，避免旋转被误判为方向

## 6. 🏥 医护功能设计

### 6.1 生命体征监测（MAX30102，独立模拟 I2C + 独立供电滤波）

| 指标 | 正常范围 | 报警阈值 | 报警动作 |
|------|---------|---------|---------|
| 心率 (HR) | 60~100 bpm | > 120 bpm 或 < 50 bpm | 语音"心率异常" + 振动间歇报警 + 蓝牙 SOS 上报 |
| 血氧 (SpO2) | 95% ~ 100% | < 90% | 语音"血氧偏低，请吸氧" + 振动长振 |

**PPG 信号处理流水线**（分散读取 → 批量处理，不阻塞主循环）：
```
[每 10ms, SoftI2C_3 @ 400kHz]
  MAX30102 FIFO 读取 1 个样本 (6 B) → ring_buf[idx++]  (耗时 ~195μs)

[每 100ms, 累积 10 个新样本触发]
  ring_buf[0..9] → DC 均值滤波 (去除基线漂移)
                 → 4 阶 Butterworth 带通 0.5~5Hz (仅保留脉搏频段)
                 → 自适应峰值检测 (动态阈值 = 均值 + 0.6×标准差)
                 → HR = 60 / ΔT_peak_avg
                 → SpO2: R_ratio = (AC_red/DC_red) / (AC_ir/DC_ir)
                 → SpO2 ≈ 104 - 17×R_ratio (经验拟合)
```

### 6.2 跌倒检测（纯软件，基于 JY61P 加速度）

```
三步判定法 (每 10ms 执行):

Step 1 — 失重检测:
  a_mag = √(ax² + ay² + az²)
  若 a_mag < 0.4g 持续 > 30ms → 疑似自由落体 → flag_freefall = 1

Step 2 — 撞击检测:
  flag_freefall == 1 状态下
  若 a_mag > 3.0g → 确认撞击 → flag_impact = 1, 记录撞击时间戳

Step 3 — 静止确认:
  flag_impact == 1 状态下
  在撞击后 10 秒内，若三轴加速度方差均 < 阈值 (静止):
    → 判定为跌倒 → 语音大音量"检测到跌倒，需要帮助吗？"
    → 启动 10 秒确认窗口: 无手势回应"取消" → 蓝牙 SOS + DFPlayer 循环呼救
```

## 6.6 v2.4 四大防御性升级详解

### 6.6.1 ★ JY61P 200Hz 速率锁定 — 根除 Jerk 一阶微分伪零点

**暗桩**：JY61P 出厂默认输出频率为 100Hz（每 10ms 一组新数据），但主循环以 200Hz（每 5ms）轮询。连续两次读到的可能是同一组旧数据，导致 Jerk 一阶微分恒等于 0，算法误判为"手静止"。

**解决**：`JY61P_Init()` 中通过 I2C 写入解锁序列 + 频率寄存器 `0x03=0x0A` (200Hz) + Flash 保存，锁定传感器以 200Hz 速率刷新数据，与 STM32 5ms 轮询严格对齐，确保相邻样本必定不同（除非手真的静止）。

### 6.6.2 ★ MAX30102 FIFO 突发清仓 — 防渐进式积压溢出

**暗桩**：v2.3 每 10ms 只读 1 个样本。主循环任何微小抖动（如 DFPlayer DMA 占用总线）都会导致 FIFO 读速 < 写速，样本渐进堆积直至溢出锁死。

**解决**：`MAX30102_ReadFIFO()` 改为 `while (unread > 0)` 循环，每次 10ms 节点一次性捞空 FIFO 中全部未读样本。单次最多读 32 样本（防独占总线），溢出时自动执行 FIFO_RESET。

### 6.6.3 ★ ADC Ping-Pong 双缓冲快照 — 删除 DMA Stop/Start 切流

**暗桩**：v2.3 在 20ms 任务中暂停 ADC DMA（`DMA_SxCR.EN=0`），读完后再恢复。高频切流存在三个风险：① EN 位清除到数据流真正停止有 2~3 个 DMA 周期的延迟；② 暂停期间丢失 ADC 采样；③ 恢复后 NDTR 可能错位导致通道映射偏移。

**解决**：ADC DMA 缓冲区扩容为 5 通道 × 2 组 = 10 半字。`HAL_ADC_ConvHalfCpltCallback`（前半组完成）和 `ConvCpltCallback`（后半组完成）各锁定一组稳定快照。`Flex_Update()` 只读取 ISR 锁定的那个稳定半组，完全不触碰 DMA_CR 寄存器，零暂停、零丢数、零撕裂。

### 6.6.4 ★ 250ms 双手异步松弛锁存窗 — 消灭生理漏判

**暗桩**：人类双手运动存在 50~200ms 的天然异步延迟。单手先完成动作时，另一手还在过渡中，`encode_fingers()` 输出的 10 指编码与词库任意条目都不匹配 → 漏判。

**解决**：手势状态机新增 `ST_LATCH` 状态。进入 ARMED 后不立即触发，而是打开 250ms 的松弛存活期。窗口内每帧持续更新手形编码并查表，窗口关闭后输出最佳匹配结果。确保双手编码在最大同步点被捕获。

```
ST_WAIT_STABLE → ST_WAIT_TRAJ → ST_ARMED → ST_LATCH(★250ms) → ST_COOLDOWN
```

## 8. 原型搭建步骤

### Step 1：CubeMX 工程搭建（0.5 天）

1. HSE (8MHz) + PLL → SYSCLK 168MHz, APB1 42MHz, APB2 84MHz
2. ADC1: 5 通道规则组 (PA0~PA4), DMA2_Stream0 循环模式, 12bit, 采样时间 480cyc
3. ADC2: 5 通道规则组 (PA5/PA6/PA7/PB0/PB1), DMA2_Stream2 循环模式
4. USART1: PA9(TX)/PA10(RX), 9600-8-N-1 (DFPlayer)，**启用 TX DMA** (DMA2_Stream7, Channel4)
5. USART3: PC10(TX)/PC11(RX), 9600-8-N-1 (蓝牙)
6. TIM3: 1ms 中断基准
7. TIM4: CH1(PD12)/CH2(PD13) PWM, 1kHz, 经 SI2302 N-MOS 低端驱动振动马达 (GPIO→Gate, 马达线圈在 Drain 端, 并联 1N4148 续流)
8. 所有软 I2C 引脚配置为 GPIO Output/Input（代码中操作，CubeMX 中仅设为 GPIO）

### Step 2：软 I2C 驱动验证（0.5 天）

- 写 `soft_i2c.c`，实现 `soft_i2c_start()`, `soft_i2c_stop()`, `soft_i2c_write_byte()`, `soft_i2c_read_byte()`
- 用逻辑分析仪/示波器验证 SCL 频率 ≈ 100kHz，SCL 上升后 ≥ 2μs 再读 SDA
- 用软 I2C 读 JY61P Roll 寄存器 (0x3D, 2 字节)，若返回 ACK 则在线
- ★ 重要: JY61P 出厂为 UART 模式，必须先用 PC 上位机配置为 I2C 模式

### Step 3：单手套硬件搭建（1 天）

- 5 个柔性传感器 + 10KΩ 分压电阻，缝/热熔胶固定到手套指背

- 排线整理：SCL / GND / SDA 三线并排（GND 夹中间！）
- 连接到面包板上的 F407

### Step 4：驱动移植 + 新增模块（1.5 天）

- 从合泰杯代码移植 `flex_sensor.c` 到 F4 HAL（ADC 读取接口适配）

- 写 `max30102.c`（软 I2C 写配置寄存器 + 读 FIFO + PPG 算法）
- 写 `dfplayer.c`：**强制使用 HAL_UART_Transmit_DMA() 或 HAL_UART_Transmit_IT() 发送 hex 指令帧，严禁阻塞式 HAL_UART_Transmit()**。DFPlayer 指令帧固定 10 字节，@9600bps = 10.4ms 阻塞；DMA 模式仅需 CPU 写入内存 → 立即返回，主循环零等待。
- 写 `vibrator.c`（TIM4 PWM 占空比控制）

### Step 5：双手对接（1 天）

- 第二只手套搭建（同 Step 3）
- ADC2 + SoftI2C_2 接入
- 双路 10 指数据同步打印验证

### Step 6：手势识别 + 医护集成（2 天）

### Step 7：语音 + 外观 + 拍摄（1.5 天）

> **9600bps 下 10 字节指令 = 10.4ms 阻塞。若用 HAL_UART_Transmit() 在主循环中同步发送，会直接跳过 2 次 JY61P 5ms 轮询窗口和 1 次 10ms 姿态解算周期。DMA 发送仅需 ~2μs（写内存 + 启动 DMA），之后 CPU 立即返回主循环，零时序污染。**
>
> CubeMX 中启用 USART1_TX DMA（DMA2_Stream7, Channel4, Memory-to-Peripheral），发送时调用 `HAL_UART_Transmit_DMA(&huart1, tx_buf, 10)` 即可。

## 附 D：软 I2C 排线布线规范（图示）

```
错误布线（串扰严重）:
  [SCL] [SDA]              ← 两线紧贴，无屏蔽

正确布线（本方案要求）:
  [SCL] [GND] [SDA] [VCC]  ← GND 夹在 SCL 和 SDA 之间
  或:
  [SCL] [GND] [SDA] [GND] [VCC] [GND] [ADC1] [GND] [ADC2] [GND] ← 多信号的完整屏蔽方案

原理: 中间 GND 线提供低阻抗回流路径，吸收 SCL 切换时的电场耦合，
      防止 SDA 在 SCL 上升沿期间产生误触发毛刺。
```
