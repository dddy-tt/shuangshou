## 1. 概述

在 v2.4 手势翻译手套基础上新增三个功能方向，不改动现有 CubeMX 配置，硬件仅增加 1 个 ESP-01S 模块。

| # | 子系统 | 链路 | 硬件改动 |
|---|--------|------|:---:|
| 1 | IoT 手势控电器 | F407 → PE0 软串口 → ESP-01S → MQTT → ESP8266 → 继电器 | 加 ESP-01S |
| 2 | 手机 Web 仪表盘 | F407 → JDY-31 BLE → 手机 Chrome | 无 |
| 3 | 手机 Web 控制页 | 手机 WiFi → MQTT（手动）| 无 |
| 4 | AI 康复训练 | F407 BLE 数据 + 手机 Web Speech API + AI API | 无 |
| 5 | 手势翻译模式 | 已有，不变 | 无 |

## 2. IoT 控制子系统

### 2.1 硬件

```
F407 PE0 ──→ ESP-01S RX  (软串口 TX-only, 9600bps)
F407 GND ──→ ESP-01S GND
F407 3.3V ─→ ESP-01S VCC

接收端 (已有):
  ESP8266 → 继电器模块 → 灯泡/风扇/插座
```

- PE0 空闲，无引脚冲突
- CubeMX 不变
- ESP-01S 用电脑 USB-TTL 预配 WiFi + MQTT Broker 地址
- 软串口 TX-only ~40 行 C，不需要 EXTI/RX 中断

### 2.2 MQTT 协议

| 手势 | Topic | Payload | 效果 |
|------|-------|---------|------|
| 控制: 灯 | `glove/light` | `ON` / `OFF` | 开关灯 |
| 控制: 风扇 | `glove/fan` | `ON` / `OFF` | 开关风扇 |
| 控制: 插座 | `glove/socket` | `ON` / `OFF` | 开关插座 |

### 2.3 模式切换

手势 `22222222220` (双手全握) 保持 3 秒 → 循环切换：
`翻译模式 ↔ 控制模式 ↔ 康复模式`

DFPlayer 语音提示当前模式。振动模式短振 (翻译) / 长振 (控制) / 双振 (康复) 区分。

### 2.4 F407 端改动

- 新增 `soft_uart.h/c`: TX-only 软串口 (PE0, 9600bps)
- `gesture.c` 词汇表新增 6 条控制类条目
- `gesture.c` 新增 `mode` 状态变量 + `Gesture_GetMode()`
- `main.c` 50ms 任务: 控制模式下手势命中 → 发 MQTT 帧而非 DFPlayer

## 3. 手机 Web 页面

### 3.1 三个独立 HTML 页面

| 页面 | 文件名 | 核心功能 |
|------|--------|---------|
| 📊 实时仪表盘 | `dashboard.html` | 手指柱状图、姿态球、心率/血氧、手势记录 |
| 🎮 设备控制 | `control.html` | 手动控灯/风扇、配网设置、手势映射编辑 |
| 🤖 AI 康复 | `rehab.html` | 引导动作、AI 实时点评、训练总结 |

每个页面底部有导航按钮，点一下切到另外两个页面。

### 3.2 技术栈

- **通信**: Web Bluetooth API (Chrome 直连 JDY-31 BLE)
- **图表**: Canvas 手绘（零依赖，无框架）
- **语音输入**: Web Speech Recognition API (Chrome 内置)
- **语音输出**: Web Speech Synthesis API (Chrome 内置)
- **AI 调用**: fetch() 直调 AI API (DeepSeek / 通义千问)
- **存储**: localStorage (保存配网设置、训练历史)

### 3.3 蓝牙数据协议 (F407 → 手机, 200ms)

```
帧头  右5指  姿态角        心率/血氧  模式  左5指  校验  帧尾
0xAA  5B    3×int16(×100)  2B        1B    5B    1B   0xBB
共 18 字节，9600bps 下 18.75ms 发完
```

## 4. AI 康复训练

### 4.1 训练流程

```
1. 用户进入 rehab.html，点"开始训练"
2. 页面发送 MODE:REHAB 到 F407
3. AI 语音播报 "请做一个握拳手势"
4. F407 每 200ms 发手指弯曲度 → 手机实时显示柱状图
5. 完成度 > 80% → AI 点评 "食指弯得不错，无名指再多弯一点"
6. 自动进入下一个动作
7. 一组训练结束 → AI 总结+建议
```

### 4.2 AI Prompt 设计

**实时指导** (每个动作后):
```
你是一位手部康复治疗师。用户正在做康复训练。
当前目标手势: 握拳
目标手指弯曲度: [80, 80, 80, 80, 80]
用户实际弯曲度: [82, 78, 85, 60, 75]
请用鼓励的语气简短点评（不超过3句话），指出差距最大的手指。
```

**训练总结** (一组结束后):
```
用户完成了一组手部康复训练。
动作序列: 握拳→伸展→拇指弯曲→OK手势→比耶
得分: [85, 92, 78, 80, 88]
历史平均: 80
心率: 72 bpm, 血氧: 98%
请给简短总结和明日训练建议。
```

### 4.3 预设训练序列

- 手指独立性: 拇指、食指、中指、无名指、小指依次伸出
- 力量训练: 握拳保持 → 完全伸展 → 重复
- 协调训练: OK 手势、比耶、单指弯曲
- 日常动作: 抓握杯子、捏东西、打字手势
